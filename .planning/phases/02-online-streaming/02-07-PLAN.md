<!-- CRITICAL: How Discord RPC and subtitle prefs get their data (addresses BLOCKER 1 and BLOCKER 2) -->

The player iframe (tau-player://) CANNOT access window.animecix (Electron IPC).
The animecix.tv website IS the bridge — it receives player postMessages and forwards
episode metadata via Electron IPC.

EXISTING behavior in animecix-angular/src/app/site/player/player.component.ts:
  - On episode change (chEffect): calls `electron.ipcRenderer.send('updateCurrent', url, identifier)`
  - On video load (activeVideoEffect): calls `electron.ipcRenderer.send('captions', video.captions)`
  - On currentTime postMessage from player: receives {time, duration, isPlaying}
  - On captionsChanged postMessage from player: updates selectedCaption

WHAT WE ADD to the preload/main IPC surface:
  1. `episode:update` (site->main): animecix.tv sends episode metadata when video changes
     - Already partially done via 'updateCurrent' IPC. We add a richer channel for Discord RPC.
  2. `episode:playState` (site->main): animecix.tv forwards isPlaying from currentTime postMessages
     - animecix.tv already receives currentTime {time, duration, isPlaying} from player every 5s
     - It should forward isPlaying to main via this IPC channel for Discord RPC updates
  3. `subtitle:get` (site->main->site): animecix.tv requests saved subtitle pref for anime
  4. `subtitle:set` (site->main): animecix.tv persists subtitle pref when player changes it

The animecix.tv Angular code already has the postMessage listener and Electron detection.
It already sends 'updateCurrent' and 'captions' IPC. The new IPC channels (episode:update,
episode:playState, subtitle:get, subtitle:set) follow the same pattern — animecix.tv
will call them when it detects the Electron environment.

NOTE: We do NOT modify animecix-angular code in this plan. We only expose the IPC channels
in preload.ts. The animecix.tv site will be updated separately to call these new channels
(it already has the pattern: `if (fromApp && electron) { electron.ipcRenderer.send(...) }`).

CRITICAL ARCHITECTURE NOTE (BLOCKER 1 + BLOCKER 2 fix):
The player iframe (tau-player://) CANNOT access window.animecix. The animecix.tv
website is the bridge:
- animecix.tv receives `currentTime` postMessages from the player (with isPlaying)
- animecix.tv receives `captionsChanged` postMessages from the player
- animecix.tv already calls `electron.ipcRenderer.send('updateCurrent', ...)` on episode change

We add new IPC channels that animecix.tv will call using the same pattern.

```typescript
import { ipcMain } from 'electron';

// Inside app.whenReady:

// --- Subtitle preference IPC (BLOCKER 2 fix) ---
// animecix.tv calls these when:
//   - Episode loads: getSubtitlePref -> gets saved lang -> sends changeSub postMessage to player
//   - Player changes subtitle: receives captionsChanged postMessage -> calls setSubtitlePref
ipcMain.handle('subtitle:get', (_event, animeId: string) => {
  return storage?.getSubtitlePref(animeId) ?? 'tr';
});
ipcMain.handle('subtitle:set', (_event, animeId: string, language: string) => {
  storage?.setSubtitlePref(animeId, language);
});

// --- Episode metadata IPC for Discord RPC (BLOCKER 1 fix) ---
// animecix.tv calls episode:update on episode change (same pattern as existing 'updateCurrent' IPC)
// animecix.tv calls episode:playState when it receives currentTime postMessage from player
//   (currentTime postMessage contains {time, duration, isPlaying} every 5s)
let lastEpisodeData: { title: string; season: number; episode: number; translator?: string; posterUrl?: string } | null = null;

ipcMain.on('episode:update', (_event, data: { title: string; season: number; episode: number; translator?: string; posterUrl?: string }) => {
  lastEpisodeData = data;
  discord?.updateActivity({ ...data, isPlaying: true, startTimestamp: Date.now() });
});

ipcMain.on('episode:playState', (_event, isPlaying: boolean) => {
  if (lastEpisodeData) {
    discord?.updateActivity({
      ...lastEpisodeData,
      isPlaying,
      startTimestamp: isPlaying ? Date.now() : undefined,
    });
  }
});

ipcMain.on('episode:idle', () => {
  lastEpisodeData = null;
  discord?.setIdle();
});
```
