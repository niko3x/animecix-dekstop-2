# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Online Streaming.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase 2 UI Surface Inventory

**CRITICAL CONTEXT:** Phase 2 adds no custom renderer UI. The two visible surfaces
are third-party-owned and must not be modified:

| Surface | Owner | Electron role |
|---------|-------|---------------|
| animecix.tv main page | Website (external) | Serve unchanged via WebContentsView |
| tau-website iframe player | Bundled (Angular/Vidstack) | Serve via `tau-player://` protocol; zero code changes |

The native surfaces Electron DOES own in this phase:

| Native Surface | What it is |
|----------------|-----------|
| Window chrome | Inherits Phase 1 frameless window — no changes |
| Deep link prompt | macOS/Windows OS-level dialog (no app UI needed) |
| Discord Rich Presence | Discord overlay — controlled via `@xhayper/discord-rpc` fields |
| Error state overlay | Electron-rendered overlay inside the window when protocol or network fails |

This spec governs the **error state overlay** (the only new Electron-rendered element)
and documents design constants for the tau-website player (read-only reference — do not override).

## Error State Overlay — Interaction Contract

This is the only new interactive Electron surface in Phase 2.

**Trigger:** `tau-player://` protocol handler returns a non-200 status or throws.

**Behavior:**
- Overlay renders full-window (`position: fixed; inset: 0`) above the main webContents
- Background: `#0a0a0a` at 95% opacity (allows user to see window is still alive underneath)
- Error card: centered, `#1a1a1a` background, `md` (16px) padding, `lg` (24px) border-radius
- Heading: 20px semibold, white — "Oynatıcı yüklenemedi"
- Body: 16px regular, `rgba(255,255,255,0.7)` — "Bağlantınızı kontrol edin ve tekrar deneyin."
- Error code: 14px regular, `rgba(255,255,255,0.4)`, monospace — "Hata kodu: [code]"
- Retry button: `sm` (8px) vertical padding, `md` (16px) horizontal padding, `#00b3ff` border (1px solid), transparent background, white text — "Tekrar Dene"
- Retry button hover: `#00b3ff` background, white text, transition 150ms ease

**Retry action:** Re-registers and re-invokes the `tau-player://` protocol handler. Does NOT reload the entire window.

## Implementation Notes for Executor

1. **Do not add CSS to tau-website.** The bundled player is a black-box artifact. All Electron-side CSS lives in `animecix-v2/src/`.
2. **Error overlay is renderer-side.** Implement as a React/DOM element injected into `animecix-v2/src/renderer.ts`, hidden by default, shown via IPC from main when the protocol fails.
3. **Skip button styling is already correct.** tau-website/src/index.css defines `.skip` — no overrides needed.
4. **Discord copy fields are string templates.** Implement them in `src/integrations/discord-rpc.ts` using the Video model fields documented in RESEARCH.md.
5. **Font loading.** Roboto is loaded by tau-website via Google Fonts inside the iframe. The Electron renderer (error overlay) uses the system font stack from `animecix-v2/src/index.css` — no additional font loading needed.
