<subtitle_data_flow>
<!-- CRITICAL: Subtitle preference sync architecture (addresses BLOCKER 2) -->

The player iframe (tau-player://) CANNOT access window.animecix (Electron IPC).
The animecix.tv website IS the bridge — it has access to both:
  1. postMessage communication with the player iframe
  2. window.animecix IPC (exposed by preload.ts)

Data flow for subtitle preferences:
  - SOURCE OF TRUTH: SQLite via StorageService (accessed through IPC)
  - ON PLAYER LOAD: animecix.tv calls window.animecix.getSubtitlePref(animeId) via IPC,
    gets saved language, sends `changeSub` postMessage to player iframe
  - PLAYER localStorage: local cache only (same tau-player:// origin across episodes),
    used for fast default when animecix.tv hasn't sent changeSub yet
  - ON SUBTITLE CHANGE IN PLAYER: player sends `captionsChanged` postMessage to
    animecix.tv, which receives it and calls window.animecix.setSubtitlePref(animeId, language)
    via IPC to persist to SQLite

The built-in player should:
  - Read localStorage('prefered_language') as a FAST DEFAULT (same as tau-website)
  - Write to localStorage on subtitle change (local cache)
  - Send `captionsChanged` postMessage to parent on every subtitle change
  - NOT attempt to call IPC directly — animecix.tv handles the SQLite bridge
</subtitle_data_flow>
