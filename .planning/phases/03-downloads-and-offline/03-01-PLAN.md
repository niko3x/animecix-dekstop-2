    1. Create `animecix-v2/src/download/Downloader.ts`:

    - Class `Downloader extends EventEmitter` with events: 'progress', 'complete', 'error', 'paused'
    - Constructor: `(url: string, outputPath: string, chunks: ChunkState[], storage: StorageService)`
    - Static method `splitIntoChunks(totalBytes: number, threadCount: number = 4): { start: number; end: number }[]` — divides totalBytes into threadCount ranges. Uses `Math.floor(totalBytes / threadCount)` for chunk size; last chunk gets remainder.
    - Method `start(): void` — starts parallel downloads for all incomplete chunks using `https.get` with `Range: bytes=${chunk.byteStart + chunk.bytesDownloaded}-${chunk.byteEnd}` header. Each chunk writes to its `tempPath` using `fs.createWriteStream({ flags: 'a' })`. On every 1 MB of data received per chunk, call `storage.updateChunkProgress(...)`. Track speed using a sliding window of last 3 seconds of bytes received. When all chunks complete, call `mergeChunks()` then emit 'complete'.
    - Method `pause(): void` — aborts all active `http.ClientRequest` objects (store references), updates status to 'paused' in storage, emits 'paused'.
    - Method `resume(): void` — calls `start()` which skips completed chunks.
    - Private method `mergeChunks(): Promise<void>` — opens `fs.createWriteStream(outputPath)`, for each chunk sorted by chunkIndex: pipe `fs.createReadStream(chunk.tempPath)` into output (with `{ end: false }`), wait for 'end' event, then `fs.unlinkSync(chunk.tempPath)`. After all chunks piped, call `output.end()`.
    - Per D-01: use native `https` module only, NO external download libraries.
    - Per D-04: 4 parallel chunks per download.
    - NEVER set `rejectUnauthorized: false` — use default TLS validation (AUTH-02).
    - Use `url.URL` to parse the download URL. Support both `http:` and `https:` schemes (use `http.get` for http URLs, `https.get` for https URLs).

    2. Create `animecix-v2/src/download/DownloadQueue.ts`:

    - Class `DownloadQueue extends EventEmitter` with events: 'progress', 'downloadComplete', 'queueEmpty', 'error'
    - Constructor: `(storage: StorageService, downloadsDir: string)` — `downloadsDir` defaults to `path.join(app.getPath('downloads'), 'AnimeciX')` per D-07. On construction, call `storage.getIncompleteDownloads()` and restore any 'downloading' items to 'queued' status (clean crash recovery). Then call `processNext()`.
    - Method `add(episodeId: string, url: string, title: string, subUrls: { language: string; url: string }[]): string` — generates id as `episodeId`, creates outputPath as `path.join(downloadsDir, sanitizeFilename(title) + '.mp4')`, calls HEAD request to get Content-Length, then calls `Downloader.splitIntoChunks(totalBytes, 4)` to create chunk definitions with tempPaths as `outputPath + '.part' + chunkIndex`, calls `storage.enqueueDownload(...)`, then calls `processNext()`. Returns the download id.
    - Method `pause(id: string): void` — if id is active download, call `downloader.pause()` and update status. If queued, just update status to 'paused'.
    - Method `resume(id: string): void` — update status to 'queued', call `processNext()`.
    - Method `cancel(id: string): void` — if active, abort downloader. Delete temp files. Call `storage.deleteDownload(id)`. Call `processNext()`.
    - Method `getQueue(): DownloadProgress[]` — return all downloads from storage mapped to DownloadProgress.
    - Private method `processNext(): void` — if there's already an active download, return (per D-03: sequential, 1 at a time). Get first 'queued' item from storage. If none, emit 'queueEmpty', return. Set status to 'downloading'. Create `new Downloader(...)`. Wire events: on 'progress' -> update storage + emit 'progress' on queue. On 'complete' -> update status to 'completed', start subtitle downloads, emit 'downloadComplete', call `processNext()`. On 'error' -> set status to 'failed', emit 'error', call `processNext()`.
    - Private method `downloadSubtitles(item: DownloadQueueItem): Promise<void>` — per D-09: for each subUrl in item.subUrls, use `https.get` to download .ass file to `path.join(path.dirname(item.outputPath), sanitizeFilename(item.title) + '.' + sub.language + '.ass')`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer -> main (IPC) | Download URLs and episode IDs from untrusted website context |
| main -> network | HTTP Range requests to external CDN (tau-video.xyz) |
| main -> filesystem | Writing downloaded chunks and merged files to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Downloader.ts URL parameter | mitigate | Validate download URL scheme is https:// before starting; reject file://, data://, javascript:// |
| T-03-02 | Tampering | DownloadQueue outputPath | mitigate | sanitizeFilename strips path separators; outputPath always constructed from downloadsDir + sanitized title, never from user input directly |
| T-03-03 | Denial of Service | Downloader totalBytes | mitigate | Enforce max download size (e.g., 10 GB) before starting; reject Content-Length > limit |
| T-03-04 | Information Disclosure | rejectUnauthorized bypass | mitigate | Never set rejectUnauthorized: false; use default TLS validation (AUTH-02 locked) |
| T-03-05 | Elevation of Privilege | StorageService SQL injection | accept | better-sqlite3 parameterized queries prevent injection; all methods use .prepare().run() with ? placeholders |
</threat_model>
